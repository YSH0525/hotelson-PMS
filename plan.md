# 현금 시재 관리 기능 구현 계획

## 개요
매일 영업 시작 시 시재금(준비금)을 설정하고, 하루 동안의 현금 입출금을 자동 집계하여, 마감 시 실제 시재와 대조하는 기능을 추가합니다.

---

## 1단계: 데이터베이스 스키마

### 새 테이블: `cash_ledger` (현금 시재 장부)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID PK | |
| entry_date | DATE | 영업일 |
| entry_type | TEXT | `opening` (시재금) / `income` (입금) / `expense` (출금) / `closing` (마감) |
| category | TEXT | `시재금`, `숙박`, `대실`, `기타매출`, `환불`, `지출`, `마감정산` 등 |
| description | TEXT | 내역 설명 |
| amount | INTEGER | 금액 (양수) |
| reservation_id | UUID? | 연결된 예약 (자동 입금 시) |
| created_by | UUID | 작성자 |
| created_at | TIMESTAMPTZ | |

**핵심 로직:**
- `opening`: 하루 시작 시재금 (1일 1건)
- `income`: 현금 결제 건별 자동 기록 + 수동 입금
- `expense`: 비품 구매, 환불 등 수동 출금
- `closing`: 마감 시 실제 보유 현금 입력 (1일 1건)

### 파일 변경
- `supabase/schema.sql` — 테이블 + RLS 정책 추가
- `src/types/database.ts` — `CashLedger` 타입 추가

## 2단계: React Query 훅

### `src/hooks/use-cash-ledger.ts`

| 훅 | 역할 |
|----|------|
| `useCashLedger(date)` | 특정 날짜의 시재 장부 전체 조회 |
| `useCashSummary(date)` | 시재금 + 현금입금합계 - 출금합계 = 예상시재 계산 |
| `useCreateCashEntry()` | 입금/출금/시재금/마감 기록 생성 |
| `useDeleteCashEntry()` | 기록 삭제 |

### 예상시재 계산 공식
```
예상시재 = 시재금(opening) + Σ입금(income) - Σ출금(expense)
과부족 = 실제시재(closing) - 예상시재
```

### KPI 카드 (4개)
1. **시재금** — 당일 영업 시작 준비금
2. **현금 입금** — 현금 결제 합계
3. **현금 출금** — 지출/환불 합계
4. **예상 시재** — 시재금 + 입금 - 출금

### 입금/출금 기록 다이얼로그
- 구분: 입금 / 출금
- 카테고리: 숙박, 대실, 기타매출, 환불, 지출, 기타
- 내역: 텍스트 입력
- 금액: 숫자 입력
- 메모: 선택 입력

### 마감 정산 영역
- 예상 시재 자동 계산 표시
- 실제 시재 입력 필드
- 과부족 자동 계산 (색상 구분: 초과=파란색, 부족=빨간색, 일치=초록색)
- 마감 확정 버튼
